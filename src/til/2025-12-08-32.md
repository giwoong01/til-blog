스택(Stack)은 컴퓨터 과학에서 데이터를 저장하고 관리하는 가장 기본적이고 중요한 선형 자료구조 중 하나입니다.

스택의 핵심 개념은 “가장 나중에 들어온 데이터가 가장 먼저 나간다”는 것입니다. 이를 LIFO (Last In, First Out)또는 후입선출 구조라고 합니다.

## 1. 스택의 개념적 모델

스택을 이해하는 가장 직관적인 비유는 ‘프링글스 통’이나 ‘쌓아 올린 접시 더미’입니다.

- 접시를 쌓을 때는 맨 위에 둡니다. (Push)
- 접시를 꺼낼 때는 맨 위에 있는 것부터 꺼내야 합니다. (Pop)
- 맨 아래에 있는 접시를 꺼내려면 위에 있는 모든 접시를 먼저 치워야 합니다.

이처럼 스택은 데이터의 입력과 출력이 Top(맨 위)이라는 한쪽 끝에서만 이루어지는 제한적인 접근 방식을 가집니다.

## 2. 스택의 주요 연산

스택은 다음과 같은 추상 자료형 연상을 정의합니다.

1. Push (삽입): 스택의 맨 위(Top)에 새로운 데이터를 추가합니다. 스택이 가득 찼는데 Push를 시도하면 Stack Overflow 에러가 발생합니다.
2. Pop (삭제): 스택의 맨 위(Top)에 있는 데이터를 꺼내서 반환하고 제거합니다. 스택이 비어있는데 Pop을 시도하면 Stack Underflow 에러가 발생합니다.
3. Peek (조회): 스택의 맨 위에 있는 데이터를 반환하되, 제거하지는 않습니다. (단순 확인용)
4. isEmpty: 스택이 비어있는지 확인합니다.
5. isFull: (고정 크기 스택의 경우) 스택이 가득 찼는지 확인합니다.

## 3. 스택의 시간 복잡도

스택은 데이터 삽입과 삭제가 특정 위치(Top)에서만 일어나므로 매우 효율적입니다.

- 삽입 (Push): O(1)
- 삭제 (Pop): O(1)
- 조회 (Peek): O(1)
- 검색 (Search): 특정 데이터를 찾으려면 위에서부터 하나씩 꺼내보며 찾아야 하므로 O(N)이 소요됩니다.

## 4. 스택의 활용 사례

스택은 “작업의 이력을 기억하고, 가장 최근 상태로 되돌아가야 하는” 모든 상황에 사용됩니다.

1. 함수 호출 스택 (Call Stack)
   - 프로그램에서 함수 A가 함수 B를 호출하면, A의 현재 상태(지역 변수, 리턴 주소 등)를 스택에 저장(Push)하고 B로 이동합니다.
   - B가 종료되면 스택에서 A의 상태를 꺼내(Pop) 원래 실행 흐름으로 복귀합니다. 재귀 함수가 동작하는 원리이기도 합니다.
2. 웹 브라우저의 ‘뒤로 가기’
   - 새로운 페이지를 방문할 때마다 주소를 스택에 Push합니다.
   - ‘뒤로 가기’를 누르면 가장 최근 주소를 Pop하여 이전 페이지로 이동합니다.
3. 편집기의 ‘실행 취소 (Undo)’
   - Ctrl + z 기능은 작업 내역을 스택에 저장했다가, 가장 최근 작업부터 역순으로 취소합니다.
4. 괄호 유효성 검사
   - 수식 ((A + B) \* C) 에서 여는 괄호 ( 가 나오면 Push, 닫는 괄호 ) 가 나오면 Pop을 하여 짝이 맞는지 검사합니다.
5. 후위 표기법 계산
   - 컴퓨터가 수식을 계산할 때 연산자 우선순위를 처리하기 위해 스택을 사용합니다.

## 5. Java에서의 스택 사용

Java에서는 java.uril.Stack 클래스를 제공하지만, 이는 오래된 레거시 클래스이므로 사용이 권장되지 않습니다.

대신 java.util.Deque 인터페이스의 구현체인 ArrayDeque를 사용하는 것이 성능상 더 유리합니다.

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class StackExample {
    public static void main(String[] args) {
        // Stack 대신 ArrayDeque 사용 권장
        Deque<Integer> stack = new ArrayDeque<>();

        // 1. Push
        stack.push(10);
        stack.push(20);
        stack.push(30);

        // 2. Peek (맨 위 확인: 30)
        System.out.println(stack.peek());

        // 3. Pop (꺼내기: 30 -> 20 -> 10)
        while (!stack.isEmpty()) {
            System.out.println(stack.pop());
        }
    }
}
```

## 정리

스택은 “후입선출(LIFO)”이라는 단순한 규칙을 가지지만, 운영체제의 메모리 관리부터 컴파일러의 수식 계산까지 컴퓨터 시스템 전반을 지탱하는 필수적인 자료구조입니다.
