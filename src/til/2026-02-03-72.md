정적 IP와 동적 IP 할당 방식의 핵심 차이는 “IP 주소 설정의 주체(관리자 vs 프로토콜)”와 “주소의 지속성(고정 vs 변동)”에 있습니다.

## 1. 정적 IP 주소 할당

네트워크 관리자나 사용자가 해당 장치의 네트워크 설정에 직접 IP 주소 정보를 입력하여 할당하는 방식입니다.

- 작동 방식
  - 사용자가 OS의 네트워크 설정 화면에서 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 주소를 직접 입력합니다.
  - 입력된 정보는 장치의 비휘발성 메모리에 저장되어, 장치를 재부팅하더라도 설정이 유지됩니다.
- 특징
  - 고정성: 설정을 변경하지 않는 한 IP 주소가 절대 변하지 않습니다.
  - 직관적 경로: 외부 장치에서 해당 IP로 접속을 시도할 때, 항상 동일한 장치로 연결됨을 보장합니다.
- 사용 목적
  - 외부의 지속적인 접속 요청을 처리해야 하는 서버(Web, DB, FTP 등)
  - 네트워크 인프라 장비(라우터, 스위치, 네트워크 프린터)의 관리 인터페이스
- 장단점
  - (장점) 접속 안정성이 높고 트러블슈팅이 용이합니다.
  - (단점) 장치 수가 많을 경우 설정에 많은 시간이 소요되며, 관리자 실수로 인한 IP 주소 중복(충돌) 가능성이 있습니다.

## 2. 동적 IP 주소 할당

DHCP(Dynamic Host Configuration Protocol)라는 네트워크 프로토콜을 사용하여, 서버가 클라이언트 장치에게 IP 주소를 자동으로 할당하는 방식입니다.

- 작동 방식
  - 클라이언트 장치가 네트워크에 연결되면 DHCP 서버(보통 라우터)에 IP 주소를 요청하는 패킷을 브로드캐스트합니다.
  - DHCP 서버는 자신이 관리하는 IP 주소 풀에서 사용되지 않는 IP를 하나 선택하여 클라이언트에게 할당합니다.
  - 이 과정은 임대 개념으로 이루어지며, 정해진 유효 시간이 지나면 갱신하거나 반납해야 합니다.
- 특징
  - 변동성: 임대 시간이 만료되거나 장치가 재부팅되면, 이전에 사용하던 IP와는 다른 IP를 할당받을 수 있습니다.
  - 자동화: 사용자가 네트워크 정보를 알 필요 없이, 케이블 연결이나 Wi-Fi 접속만으로 네트워크 구성이 완료됩니다.
- 사용 목적
  - 일반 사용자용 PC, 노트북, 스마트폰, 템플릿
  - IP가 고정될 필요가 없는 대부분의 IoT 기기
- 장단점
  - (장점) 네트워크 관리가 자동화되어 효율적이며, 한정된 IP 자원을 여러 장치가 시분할하여 사용할 수 있습니다.
  - (단점) IP가 수시로 변경되므로, 외부에서 해당 장치를 특정하여 접속하기 어렵습니다. (DDNS 등 추가 설정 필요)

## 정리

| 구분 | 정적 IP 할당 (Static) | 동적 IP 할당 (Dynamic) |
| --- | --- | --- |
| 설정 주체 | 관리자 (수동 입력) | DHCP 서버 (프로토콜에 의한 자동 할당) |
| 지속성 | 영구적 (변경 전까지 고정) | 일시적 (임대 기간 만료/재부팅 시 변경 가능) |
| 관리 효율 | 낮음 (장비별 개별 설정 필요) | 높음 (중앙 서버에서 일괄 관리) |
| IP 충돌 위험 | 있음 (관리자 실수 가능성) | 없음 (서버가 중복 체크 후 할당) |
| 주요 용도 | 서버, 네트워크 장비 | 클라이언트 단말기 (PC, 모바일) |
