CDN(Content Delivery Network, 콘텐츠 전송 네트워크)은 지리적으로 분산된 여러 대의 서버를 연결하여, 사용자에게 웹 콘텐츠(이미지, 동영상, 스크립트 파일 등)를 빠르고 안정적으로 전송하는 기술입니다.

쉽게 말해, “원본 서버의 데이터를 전 세계 곳곳에 있는 캐시 서버에 미리 복사해 두고, 사용자와 가장 가까운 서버에서 데이터를 응답해 주는 시스템”입니다.

## 1. 도입 목적

인터넷 통신도 결국 물리적인 케이블을 타고 이동하므로 거리에 비례하여 데이터 전송 지연 시간이 발생합니다.

- CDN 미적용: 한국에 있는 사용자가 미국에 위치한 원본 서버에 접속하여 10MB짜리 이미지를 다운로드 하려면, 데이터가 태평양을 건너갔다 와야 하므로 로딩 속도가 느립니다.
- CDD 적용: 한국에 위치한 CDN 서버가 미리 복사해 둔 이미지를 대신 전달해주므로, 물리적 거리가 단축되어 로딩 속도가 비약적으로 빨라집니다.

## 2. 동작 원리

1. 원본 서버: 실제 웹사이트의 원본 데이터가 저장되어 있는 메인 서버입니다. (예: AWS, EC2, S3)
2. 엣지 서버: 전 세계 주요 네트워크 거점에 흩어져 있는 CDN의 캐시 서버들입니다.
3. 캐싱: 특정 지역의 사용자가 처음 데이터를 요청하면, 해당 지역의 엣지 서버는 원본 서버에서 데이터를 가져와 사용자에게 전달하고 자신의 디스크에 복사본을 임시 저장합니다.
4. 빠른 응답: 이후 같은 지역의 다른 사용자가 동일한 데이터를 요청하면, 엣지 서버는 원본 서버까지 갈 필요 없이 자신이 가진 복사본을 즉시 반환합니다.

## 3. 장점

1. 압도적인 응답 속도 향상

   물리적으로 가까운 서버에서 응답하므로 페이지 로딩 속도가 크게 개선됩니다. 이는 사용자 경험을 향상시키고 검색 엔진 최적화에도 긍정적인 영향을 미칩니다.

1. 서버 트래픽 부하 분산 및 비용 절감

   갑자기 이벤트 등으로 트래픽이 폭주하더라도, 수많은 CDN 엣지 서버들이 트래픽을 나누어 처리합니다. 결과적으로 원본 서버로 향하는 부하가 대폭 줄어들어 서버 다운을 방지하고, 메인 서버의 아웃바운드 네트워크 대역폭 비용을 절감할 수 있습니다.

1. 보안 강화 (DDoS 방어)

   대규모 트래픽을 처리하도록 설계된 글로벌 CDN은 악의적인 DDoS 공격이나 비정상적인 접근을 엣지 서버 단에서 1차적으로 흡수하고 차단하여 원본 서버를 안전하게 보호합니다.

## 정리

CDN은 실시간으로 변하는 데이터보다는, 한 번 만들어지면 잘 변하지 않는 정적 자원을 배포하는 데 최적화 되어 있습니다.

- 고해상도 이미지, 폰트 파일
- 동영상 스트리밍 (VOD, 라이브 방송)
- HTML, CSS, JavaScript 파일
