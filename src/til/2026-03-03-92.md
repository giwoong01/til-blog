try-with-resources는 자바 7부터 도입된 문법으로, 사용이 끝난 외부 자원(파일, 네트워크 소켓, 데이터베이스 커넥션 등)을 자동으로 해제해 주는 구문입니다.

과거에 자원을 반납하기 위해 필수적으로 작성해야 했던 복잡한 finally 블록을 대체하며, 메모리 누수를 방지하고 예외 처리의 안정성을 높이는 데 목적이 있습니다.

## 1. 코드 구조

전통적인 방식과 try-with-resources 방식을 코드로 비교해보면 차이가 명확합니다.

### 기존 방식

자원을 해제하기 위해 finally 블록에서 close()를 직접 호출해야 합니다. close() 자체도 예외를 던질 수 있어 try-catch가 중첩되어 코드가 지저분해집니다.

```java
FileInputStream fis = null;
try {
    fis = new FileInputStream("file.txt");
    // 파일 읽기 로직
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close(); // 자원 반납
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### try-with-resources 방식

try(…) 괄호 안에 사용할 자원을 선언합니다. try 블록의 실행이 끝나면(정상 종료든, 예외 발생이든 상관없이) 자동으로 close() 메소드가 호출됩니다.

```java
// 괄호 안에 객체 생성 및 할당
try (FileInputStream fis = new FileInputStream("file.txt")) {
    // 파일 읽기 로직
} catch (IOException e) {
    e.printStackTrace();
}
// finally 블록이 없어도 알아서 fis.close()가 실행됨
```

## 2. 동작 원리 및 조건

- try() 괄호 안에 선언되는 객체는 반드시 java.lang.AutoCloseable 또는 java.io.Closeable 인터페이스를 구현하고 있어야 합니다.
- 자바의 표준 입출력 클래스, JDBC 커넥션 등 해제가 필요한 대부분의 라이브러리들은 이미 이 인터페이스를 구현하도록 업데이트되어 있습니다.

```java
// AutoCloseable 인터페이스 구조
public interface AutoCloseable {
    void close() throws Exception;
}
```

## 3. 장점

1. 예외 누락 문제 해결

   기존 try-finally 방식의 가장 큰 문제점은 예외가 덮어씌워지는 현상이었습니다.
   - try 블록 안에서 핵심 로직을 수행하다가 에러(A)가 터집니다.
   - finally 블록으로 넘어가서 close()를 하다가 또 에러(B)가 터집니다.
   - 이 경우 콘솔이나 로그에는 나중에 발생한 에러(B)만 남고, 진짜 원인인 에러(A)는 사라져 버립니다.

   try-with-resources는 이 문제를 “억제된 예외”라는 개념으로 해결합니다.
   - 동일한 상황에서 에러 A와 에러 B가 모두 발생하면, 진짜 원인인 에러 A를 발생시킵니다.
   - 그리고 close() 에서 발생한 에러 B는 버리지 않고, 에러 A의 Suppressed 속성에 담아서 함께 로그인에 남겨줍니다.

1. 안전한 다중 자원 해제

   여러 개의 자원을 사용할 때, 세미콜론으로 구분하여 선언할 수 있습니다. 해제는 선언의 역순으로 안전하게 진행됩니다.

   ```java
   try (
       FileInputStream fis = new FileInputStream("input.txt");
       FileOutputStream fos = new FileOutputStream("output.txt")
   ) {
       // ... 로직
   }
   // fos.close() 실행 후 fis.close() 실행 보장
   ```

## 정리

try-with-resources는 자원 반납을 JVM에게 위임하여 코드의 가독성을 높이고, 자원 누수를 완벽하게 차단하며, 에러 발생 시 최초 원인 예외가 묻히는 현상을 방지하는 모던 자바의 필수 문법입니다.
