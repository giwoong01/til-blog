Call By Value(값에 의한 호출)와 Call By Reference(참조에 의한 호출)는 함수(메소드)를 호출할 때 인자를 어떻게 전달하느냐에 따른 방식의 차이입니다.

## 1. Call By Value (값에 의한 호출)

함수를 호출할 때, 변수의 값을 복사해서 함수 인자로 전달하는 방식입니다.

- 동작 원리
  1. 함수가 호출되면, 전달받은 값을 저장할 새로운 메모리 공간(매개변수)이 생성됩니다.
  2. 원본 변수의 값을 복사하여 이 새로운 공간에 넣습니다.
  3. 함수 안에서 인자의 값을 아무리 변경해도, 원본 변수에는 아무런 영향이 없습니다.
- 장점
  - 원본 데이터가 보호되어 안전합니다.
- 단점
  - 데이터 크기가 크면(거대한 구조체 등) 복사하는데 메모리와 시간이 많이 소모됩니다.

## 2. Call By Reference (참조에 의한 호출)

함수를 호출할 때, 변수의 메모리 주소(참조)를 함수 인자로 전달하는 방식입니다.

- 동작 원리
  1. 함수의 매개변수는 새로운 공간을 잡는 대신, 원본 변수의 주소를 저장합니다.
  2. 함수 내부에서 매개변수를 조작하면, 주소를 타고 들어가 원본 변수의 데이터를 직접 변경합니다.
- 장점
  - 데이터를 복사하지 않으므로 빠르고 메모리를 절약합니다.
- 단점
  - 함수 실행 결과가 원본 데이터에 영향을 미치므로, 데이터 관리에 주의합니다.

## 예제

두 변수의 값을 바꾸는 swap 함수를 예로 들면 차이가 명확합니다.

Call By Value (값만 복사됨)

```c
void swap(int a, int b) {
	int temp = a;
	a = b;
	b = temp;
}
```

Call By Reference (주소를 전달함)

```c
void swap (int *a, int *b) {
	int temp = *a;
	*a = *b;
	*b = temp;
}
```

## Java의 경우에는?

면접에서 자주 나오는 질문입니다. “Java는 Call By Reference가 있는가?"

정답은 “아니오, Java는 오직 Call By Value만 존재합니다.”

하지만 객체를 전달할 때는 마치 Call By Reference 처럼 동작하는 것처럼 보여서 헷갈리기 쉽습니다.

1. 기본 자료형 (int, double 등)
   - 값이 복사되어 전달됩니다. (전형적인 Call By Value)
2. 참조 자료형 (Object, Array 등)
   - 객체의 “주소 값”이 복사되어 전달됩니다.
   - 함수 안에서 obj.setName(”Park”) 처럼 내부 필드를 바꾸면 원본 객체도 바뀝니다. (주소를 복사해 왔으니까 같은 집을 가리킴)
   - 하지만 obj = new Person(”Kim”) 처럼 변수 자체에 새로운 객체를 할당해도, 원본 변수는 변하지 않습니다. (복사된 주소표만 새 집 주소로 바뀐 것이지, 원본 주소표는 그대로임)
