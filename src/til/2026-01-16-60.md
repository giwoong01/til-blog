시스템 콜은 운영체제의 커널이 제공하는 서비스에 대해, 응용 프로그램이 이용할 수 있도록 열어둔 인터페이스를 말합니다.

쉽게 말해, “프로그램이 운영체제에게 ‘이것좀 해줘’ 라고 부탁하는 공식적인 창구”입니다.

## 1. 필요한 이유

컴퓨터 시스템의 안전성을 위해 운영체제는 이중 모드로 실행됩니다.

1. 사용자 모드
   - 우리가 실행하는 일반적인 프로그램(브라우저, 게임, 한글 등)이 실행되는 영역입니다.
   - 이곳에서 하드웨어(디스크, 메모리, 네트워크 등)에 직접 접근할 수 있는 권한이 ㅇ벗습니다. (잘못 건드려서 시스템을 망가뜨리는 것을 방지하기 위함)
2. 커널 모드
   - 운영체제 핵심부(커널)가 실행되는 영역입니다.
   - 모든 하드웨어 제어권과 CPU의 특권 명령을 실행할 수 있는 ‘신’의 권한을 가집니다.

문제: 사용자 프로그램이 파일을 저장하려면 하드 디스크에 접근해야 하는데, 사용자 모드에서는 접근 권한이 없습니다.

해결: 프로그램이 직접 디스크를 건드리는 대신, 시스템 콜을 통해 운영체제에게 “파일 좀 저장해 줘”라고 요청하면, 운영체제가 권한을 확인한 뒤 대신 처리해 주는 것입니다.

## 2. 동작 과정 (실행 흐름)

1. 요청: 사용자 프로그램이 printf() 같은 라이브러리 함수를 호출합니다.
2. 트랩 발생: 라이브러리는 내부적으로 해당 기능에 맞는 시스템 콜 번호를 레지스터에 담고, 인터럽트를 발생시켜 CPU에게 신호를 보냅니다.
3. 모드 전환: CPU는 실행 모드를 커널 모드로 바꿉니다.
4. 실행: 커널은 IDT를 참조하여 요청받은 시스템 콜 함수를 실행하여 하드웨어를 제어합니다.
5. 복귀: 작업이 끝나면 결과를 반환하고, 다시 사용자 모드로 전환하여 프로그램의 다음 코드를 실행합니다.

## 3. 시스템 콜과 성능

시스템 콜은 공짜가 아닙니다.

시스템 콜이 호출될 때마다 User Mode ↔ Kernerl Mode 간의 문맥 교환이 발생합니다. 이 과정에서 레지스터 상태를 저장하고 복구하는 비용이 들기 때문에, 시스템 콜을 너무 빈번하게 호출하면 프로그램 성능이 저하될 수 있습니다.

따라서 고성능 애플리케이션을 짤 때는 시스템 콜 호출 횟수를 줄이는 것(예: 버퍼링 사용)이 중요한 최적화 기법 중 하나입니다.
