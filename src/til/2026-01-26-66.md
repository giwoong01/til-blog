NAT(Network Address Translation, 네트워크 주소 변환)를 사용하는 이유는 크게 “주소 절약”과 “보안” 두 가지로 요약할 수 있습니다.

NAT는 라우터(공유기)가 내부 사설 IP 주소(private IP)를 외부 공인 IP 주소(public IP)로 변환해 주는 기술입니다.

## 1. IPv4 주소 고갈 문제 해결

인터넷이 폭발적으로 성장하면서, 전 세계적으로 할당 가능한 공인 IP 주소(IPv4)가 부족해졌습니다. (IPv4는 약 43억 개인데, 현재 스마트폰, PC, IoT 기기는 그보다 훨씬 많음)

- 해결책: 모든 기기에 귀한 공인 IP를 하나씩 주는 대신, 공유기 하나만 공인 IP를 사용하고, 공유기 아래에 연결된 수십, 수백 대의 기기는 사설 IP(192.168.x.x 등)를 쓰게 합니다.
- 효과: 공인 IP 1개만 있으면 수백 대의 컴퓨터가 동시에 인터넷을 할 수 있어 주소를 획기적으로 절약할 수 있습니다.

## 2. 보안 강화 (내부 네트워크 보호)

NAT를 사용하면 외부(인터넷)에서는 우리 집 내부의 컴퓨터 IP(사설 IP)를 알 수 없습니다. 외부 세상은 오직 공유기의 공인 IP만 볼 수 있습니다.

- 은닉 효과: 외부 해커나 악성 봇이 내부 PC의 정확한 주소를 모르기 때문에 직접적인 접근이나 공격이 어려워집니다.
- 방화벽 역할: 공유기(NAT 장비)가 일차적인 방어벽 역할을 하여, 허용되지 않은 외부의 접근을 차단합니다.

## 3. 비용 절감 및 관리의 유연성

- 비용: ISP(인터넷 통신사)로부터 공인 IP를 여러 개 할당받으려면 비용이 많이 듭니다. NAT를 쓰면 공인 IP 하나 값만 내면 됩니다.
- 유연성: 인터넷 통신사를 바꾸거나 공인 IP가 변경되더라도, 내부 네트워크(사설 IP 대역) 설정은 바꿀 필요가 없습니다. 내부망 구조를 외부에 의존하지 않고 독립적으로 관리할 수 있습니다.

## PAT (Port Address Translation)

우리가 흔히 쓰는 “공인 IP 1개로 여러 명이 인터넷 하기” 기술은 NAT의 일종인 PAT, 혹은 NAPT라고 부릅니다.

- 공인 IP는 하나뿐인데 컴퓨터 A와 B를 어떻게 구분할까요?
- 바로 포트 번호를 다르게 매겨서 구분합니다.
  - 컴퓨터 A의 요청 → 공인 IP: 1001번 포트로 변환
  - 컴퓨터 B의 요청 → 공인 IP: 1002번 포트로 변환
  - 응답이 오면 포트 번호를 보고 다시 주인에게 찾아줍니다.

## 정리

NAT는 “부족한 IP 주소를 아껴 쓰기 위해(경제성)” 탄생했고, 덤으로 “내부 IP를 숨겨주는(보안)”효과까지 얻게 된 필수 네트워크 기술입니다.
