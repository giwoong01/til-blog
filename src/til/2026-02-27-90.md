시간 복잡도와 공간 복잡도는 알고리즘의 성능과 효율성을 평가하는 두 가지 핵심 지표입니다.

두 개념 모두 입력 데이터의 크기(N)가 커짐에 따라 자원(시간 또는 메모리)의 소모량이 어떻게 증가하는지를 수학적으러 나타내며, 주로 빅오 표기법(Big-O Notation)을 사용하여 표현합니다.

## 1. 시간 복잡도

“알고리즘이 실행을 완료할 때까지 얼마나 많은 연산을 수행하는가?”를 나타냅니다.

- 측정 기준: 실제 초 단위의 실행 시간이 아니라, 기본 연산(비교, 할당, 사칙연산 등)의 실행 횟수를 기준으로 계산합니다.
- 주요 관심사: 입력 크기 N이 무한대로 커질 때, 연산 횟수가 어떻게 증가하는가?
- 예시
  - 1차원 배열을 한 번 순회하며 값을 찾는 선형 탐색: O(N)
  - 이중 for문을 사용하여 모든 쌍을 비교하는 버블 정렬: O(N^2)
  - 입력 데이터의 절반씩을 버리며 탐색하는 이진 탐색: O(log N)

## 2. 공간 복잡도

“알고리즘이 실행을 완료할 때까지 얼마나 많은 메모리(RAM)를 차지하는가?”를 나타냅니다.

- 측정 기준: 프로그램 실행 시 필요한 메모리 공간의 총량을 기준으로 합니다.
- 구성 요소
  - 고정 공간: 코드 자체, 전역 변수, 상수 등 입력 크기와 무관하게 항상 일정한 메모리
  - 가변 공간 (핵심): 동적으로 할당되는 배열, 객체, 그리고 재귀 함수 호출로 인해 쌓이는 콜 스택(Call Stack) 등 입력 크기 N에 따라 늘어나는 메모리
- 예시
  - 크기가 N인 1차원 배열을 새로 생성하여 저장: O(N)
  - N X N 크기의 2차원 배열(행렬)을 생성하여 저장: O(N^2)
  - 입력 크기와 상관없이 정수형 변수 몇 개만 사용하여 처리: O(1)

## 3. 시간과 공간의 트레이드오프

대부분의 알고리즘 설계에서는 시간 복잡도와 공간 복잡도 간의 반비례 관계(Trade-off)가 발생합니다. 즉, 실행 속도를 높이려면 메모리를 더 많이 써야 하고, 메모리를 아끼려면 실행 속도가 느려지는 경우가 많습니다.

- 메모리를 희생하여 시간을 단축하는 예
  - 동적 계획법(DP, Memoization): 한 번 계산한 결과를 메모리(배열 등)에 저장해 두고, 나중에 동일한 꼐산이 필요할 때 메모리에서 바로 꺼내 써서 연산 시간을 획기적으로 줄입니다.
  - 해시 테이블(Hash Table): 데이터 저장 공간을 넉넉히 할당하여, 탐색 시간을 O(1) 수준으로 끌어 올립니다.

## 정리

| 구분        | 시간 복잡도                           | 공간 복잡도                                     |
| ----------- | ------------------------------------- | ----------------------------------------------- |
| 정의        | 입력 크기에 따른 연산 횟수의 증가율   | 입력 크기에 따른 메모리 사용량의 증가율         |
| 최적화 목표 | 연산 단계를 줄여 CPU 점유 시간 최소화 | 불필요한 변수/자료구조를 줄여 RAM 사용량 최소화 |
