페이지 교체 알고리즘은 운영체제의 가상 메모리 관리에서 매우 중요한 역할을 하는 기술입니다.

## 1. 왜 필요할까요?

프로그램이 실행되려면 메모리에 올라와야 하지만, 메모리 용량은 한정되어 있습니다. 그래서 운영체제는 당장 필요한 데이터만 메모리에 올리고, 나머지는 디스크에 둡니다.

- 페이지 부재(Page Fault): CPU가 필요로 하는 데이터가 현재 메모리에 없을 때 발생하는 현상입니다.
- 이때 운영체제는 디스크에서 해당 데이터를 찾아 메모리의 빈 공간에 올려야 하는데, 메모리가 이미 꽉 차있다면 교체해야 합니다. 이때 쫓아낼 희생양을 고르는 기준이 바로 페이지 교체 알고리즘입니다.

## 2. 4가지 알고리즘

누구를 꽃아내야 시스템의 성능(페이지 부재 발생 비율 최소화)이 가장 좋아질지 다양한 전략이 존재합니다.

1. FIFO (First In First Out) 알고리즘
   - 전략: 메모리에 가장 먼저 들어온 페이지를 쫓아냅니다. (오래된)
   - 특징: 구현이 매우 간단합니다. 들어온 시간을 큐로 관리하면 됩니다.
   - 단점: 처음부터 끝까지 자주 쓰이는 중요한 데이터임에도 불구하고, 단순히 제일 먼저 들어왔다는 이유만으로 쫓겨날 수 있습니다. (할당된 페이지 프레임을 늘려주었는데 오히려 페이지 부재가 더 많이 발생하는 Belady의 모순 현상이 발생하기도 합니다.)
2. OPT (Optimal, 최적 교체) 알고리즘
   - 전략: 앞으로 가장 오랫동안 사용되지 않을 페이지를 찾아 쫓아냅니다.
   - 특징: 모든 알고리즘 중 가장 낮은 페이지 부재율을 보장하는 최고의 방법입니다.
   - 단점: 구현이 불가능합니다. 운영체제가 미래에 어떤 데이터가 쓰일지 미리 예측할 수 없기 때문입니다. 따라서 다른 알고리즘들의 성능을 평가하는 이론적인 척도로만 사용됩니다.
3. LRU (Least Recently Used) 알고리즘
   - 전략: 최근에 가장 오랫동안 사용되지 않은 페이지를 쫓아냅니다.
   - 특징: “과거를 보면 미래를 알 수 있다”는 아이디어에서 출발했습니다. 최근에 안 썼다면 앞으로도 안 쓸 확률이 높다고 판단하는 것입니다.
   - 실무적 의의: 실질적으로 최적(OPT) 알고리즘과 가장 비슷한 성능을 내기 때문에, 운영체제뿐만 아니라 웹 캐시, DB 캐시(Redis 등) 등 실무에서 가장 널리 쓰이는 표준적인 알고리즘입니다.
4. LFU (Least Frequently Used) 알고리즘
   - 전략: 과거에 참조된 횟수가 가장 적은 페이지를 쫓아냅니다.
   - 특징: 사용 빈도를 기준으로 하므로, 방금 막 메모리에 올라온 최신 페이지가 횟수가 적다는 이유로 쫓겨날 위험이 있습니다.

| **알고리즘** | **교체 기준 (희생양)**           | **장점**                          | **단점**                            |
| ------------ | -------------------------------- | --------------------------------- | ----------------------------------- |
| **FIFO**     | 가장 오래전에 메모리에 올라온 것 | 이해와 구현이 매우 쉬움           | 자주 쓰이는 것도 오래되면 쫓겨남    |
| **OPT**      | 미래에 가장 오랫동안 안 쓰일 것  | 가장 이상적이고 완벽한 성능       | 미래를 알 수 없어 **구현 불가**     |
| **LRU**      | 과거에 가장 오랫동안 안 쓰인 것  | 실질적인 최고 성능 (가장 많이 씀) | 쫓아낼 놈을 찾기 위한 오버헤드 존재 |
| **LFU**      | 지금까지 가장 적게 사용된 것     | 활발히 쓰이는 페이지 보호 가능    | 막 들어온 페이지가 불리함           |

## 정리

페이지 교체 알고리즘은 꽉 찬 메모리에서 “누구를 희생시켜야 시스템이 덜 버벅거릴까?”를 결정하는 두뇌입니다. 이론적으로는 OPT가 최고지만 불가능하므로, 현실에서는 과거의 기록을 바탕으로 판단하는 LRU 알고리즘이 가장 대중적으로 사용됩니다.
