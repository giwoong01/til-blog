네트워크에서 회선 교환 방식과 패킷 교환 방식의 가장 큰 차이점은 “통신을 시작하기 전에 물리적/논리적인 전용 통로를 미리 확보하는가, 아니면 데이터를 잘게 쪼개어 공유망으로 독립적으로 전송하는가”에 있습니다.

## 1. 회선 교환 방식

통신하는 두 장치 사이에 전용 통신 경로(회선)를 미리 설정하고, 통신이 끝날 때까지 해당 경로를 독점하여 데이터를 주고받는 방식입니다.

- 작동 방식
  1. 회선 설정: 데이터를 보내기 전에 목적지까지의 경로를 탐색하고 연결을 확립합니다.
  2. 데이터 전송: 설정된 전용 경로를 통해 데이터를 연속적으로 전송합니다.
  3. 회선 해제: 통신이 완료되면 경로를 해제하여 다른 기기가 사용할 수 있게 합니다.
- 특징
  - 경로 독점: 한 번 연결되면 통신 중에는 다른 기기가 이 경로를 사용할 수 없습니다.
  - 고정된 대역폭: 할당된 통신 속도와 대역폭이 일정하게 유지됩니다.
- 장단점
  - (장점) 전송 지연이 거의 없고, 데이터가 전송된 순서대로 정확하게 도착하므로 실시간 통신(전화망, 음성 통신)에 적합합니다.
  - (단점) 통신 경로를 독점하므로 데이터를 보내지 않는 쉬는 시간에도 회선을 점유하여 네트워크 자원의 낭비가 심합니다.

## 2. 패킷 교환 방식

전송하려는 데이터를 패킷이라는 작은 단위로 쪼갠 후, 각 패킷에 목적지 주소를 붙여 네트워크에 흘려보내는 방식입니다.

- 작동 방식
  1. 데이터를 일정한 크기의 패킷으로 분할합니다.
  2. 각 패킷은 헤더(목적지 IP 주소, 순서 번호 등)를 포함하여 독립적으로 라우터들을 거쳐 목적지로 향합니다.
  3. 목적지에 도착한 패킷들은 헤더의 순서 번호를 바탕으로 원래의 데이터로 재조립됩니다.
- 특징
  - 경로 공유: 전용 회선 없이 여러 기기의 패킷들이 동일한 네트워크 회선을 공유하여 이동합니다.
  - 비연결성/독립성: 패킷마다 네트워크 상황에 따라 최적의 경로를 찾아가므로, 같은 데이터의 패킷이라도 이동 경로가 다를 수 있고 도착 순서가 뒤섞일 수 있습니다.
- 장단점
  - (장점) 회선을 여러 사용자가 공유하므로 네트워크 자원을 매우 효율적으로 사용할 수 있으며, 특정 라우터에 장애가 생겨도 우회 경로를 통해 전송이 가능한 뛰어난 안정성(복원력)을 가집니다. (인터넷 통신의 기본 방식입니다.)
  - (단점) 각 라우터에서 패킷을 저장하고 어디로 보낼지 처리하는 시간(큐잉 지연)이 발생하며, 트랲기이 몰리면 지연 시간이 길어지거나 패킷 손실이 발생할 수 있습니다.

## 정리

| 구분        | 회선 교환                       | 패킷 교환                             |
| ----------- | ------------------------------- | ------------------------------------- |
| 핵심 개념   | 전용 경로 확립 및 독점          | 데이터를 쪼개어 경로 공유             |
| 전송 단위   | 연속적인 데이터 스트림          | 독립적인 패킷                         |
| 대역폭/자원 | 고정 할당 (자원 낭비 발생 가능) | 동적 할당 (자원 효율성 높음)          |
| 경로 유연성 | 한 번 설정되면 변경 불가        | 라우팅 상황에 따라 유연하게 우회 가능 |
| 데이터 순서 | 보낸 순서대로 정확히 도착       | 섞일 수 있음 (목적지에서 재조립 필요) |
| 주요 사용처 | 전통적인 유선 전화망            | 인터넷 (TCP/IP), 컴퓨터 네트워크      |
