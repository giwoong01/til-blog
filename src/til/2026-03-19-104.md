가상화란 단일한 물리적 하드웨어(서버, 스토리지, 네트워크 등)를 소프트웨어적으로 여러 개의 논리적 환경으로 쪼개어 사용하는 기술을 말합니다.

과거에는 하나의 서버에 하나의 운영체제만 설치해서 사용해야 했기 때문에 하드웨어의 성능을 100% 활용하기 어려웠습니다. 가상화는 이러한 물리적 한계를 극복하고 자우너 효율성을 극대화하기 위해 등장했습니다.

## 1. 하이퍼바이저

물리적 하드웨어 위에서 여러 개의 가상 머신을 만들고 관리해 주는 핵심 소프트웨어를 하이퍼바이저 또는 VMM(Virtual Machine Monitor)이라고 부릅니다.

하이퍼바이저는 하드웨어와 가상 머신 사이에 위치하여, 각 VM이 CPU, 메모리, 저장 장치 등의 자원을 독립적으로 할당받아 마치 자신이 실제 컴퓨터인 것처럼 착각하게 만듭니다.

하이퍼바이저는 설치 위치에 따라 크게 두 가지로 나뉩니다.

1. 베어메탈 / 네이티브 방식
    - 구조: 운영체제(OS) 없이 하드웨어 위에 직접 하이퍼바이저를 설치하고, 그 위에 가상 머신들을 올리는 방식입니다.
    - 특징: 중간에 거치는 OS가 없으므로 오버헤드가 적고 성능과 안정성이 매우 뛰어납니다.
    - 용도: 엔터프라이즈 환경의 대규모 서버나 클라우드 데이터센턴에서 주로 사용합니다.

1. 호스트 방식
    - 구조: 기존 하드웨어에 윈도우나 맥 같은 운영체제(Host OS)를 먼저 설치하고, 그 OS 위에서 실행되는 일반 프로그램처럼 하이퍼바이저를 설치하는 방식입니다.
    - 특징: 기존 PC 환경에서 손쉽게 설치할 수 있지만, Host OS를 한 번 거쳐야 하므로 1번에 비해 성능 손실이 발생합니다.
    - 용도: 개인 PC에서 다른 OS를 테스트하거나 개발 환경을 구축할 때 주로 사용합니다.

## 2. 가상화 도입의 장점

1. 자원 효율성 극대화 (비용 절감)
    - 서버 한 대의 성능이 남아도는 현상을 방지합니다. 여러 서버의 역할을 강력한 물리 서버 한 대에 통합하여 하드웨어 구매 비용, 전력 소모, 상면 공간(서버실 크기)을 크게 줄일 수 있습니다.
2. 독립성 및 안전성 보장
    - 각 가상 머신은 철저하게 격리되어 있습니다. 따라서 하나의 VM에 바이러스가 걸리거나 시스템이 다운되더라도, 같은 물리 서버에서 돌아가고 있는 다른 VM에는 전혀 영향을 주지 않습니다.
3. 뛰어난 유연성과 빠른 배포
    - 물리적인 서버를 주문하고 조립할 필요 없이, 마우스 클릭 몇 번이나 코드를 통해 수 분 만에 새로운 가상 서버를 만들어낼 수 있습니다. 또한, VM 자체가 하나의 파일 형태로 관리되므로 백업과 다른 서버로의 이전이 매우 쉽습니다.

## 정리

가상화라는 강력한 컴퓨터 한 대를 여러 대의 독립적인 컴퓨터처럼 쪼개어 쓰는 마법 같은 기술이며, 이 마법을 부리는 핵심 프로그램이 하이퍼바잊입니다. 오늘날 우리가 사용하는 AWS, GCP 같은 클라우드 컴퓨팅 서비스는 모두 이 가상화 기술을 기반으로 구축되어 있습니다.
