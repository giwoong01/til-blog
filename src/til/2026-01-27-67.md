연결 리스트는 데이터를 감싼 노드들이 링크(포인터)를 통해 기차처럼 줄줄이 연결된 형태의 선형 자료구조입니다.

배열과 함께 가장 기초적이고 중요한 자료구조이며, 배열의 단점을 해결하기 위해 고안되었습니다.

## 1. 구조와 작동 원리

### 핵심 구성 요소

1. 노드 (Node): 데이터를 담는 그릇입니다. 두 가지 정보를 가집니다.
   - 데이터: 실제 저장하려는 값 (예: 숫자 10, 문자 “A”)
   - 포인터: 다음 노드가 메모리 어디에 있는지 가리키는 주소값
2. 헤드 (Head): 리스트의 가장 첫 번째 노드를 가리키는 포인터입니다. 헤드를 잃어버리면 리스트 전체를 잃어버리는 것과 같습니다.
3. 테일 (Tail): 리스트의 가장 마지막 노드입니다. 다음을 가리키는 포인터가 Null(없음)입니다.

## 2. 배열과의 차이

| 특징               | 배열                                              | 연결 리스트                            |
| ------------------ | ------------------------------------------------- | -------------------------------------- |
| 메모리 구조        | 연속적 (붙어있음)                                 | 비연속적 (여기저기 흩어져 있음)        |
| 데이터 접근 (검색) | 빠름 O(1)                                         | 느림 O(N)                              |
|                    | 인덱스로 바로 접근 가능                           | 첫 노드부터 순서대로 따라가야 함       |
| 데이터 추가/삭제   | 느림 O(N)                                         | 빠름 O(1)                              |
|                    | 중간 데이터를 지우면 뒤의 데이터를 다 당겨와야 함 | 앞뒤 노드의 연결(포인터)만 바꿔주면 됨 |
| 크기               | 고정 (처음에 정해야 함)                           | 가변 (필요할 때마다 늘리고 줄임)       |

## 3. 연결 리스트의 종류

1. 단일 연결 리스트
   - 특징: 한 방향으로만 연결됩니다. (Next만 있음)
   - 단점: 뒤로 돌아갈 수 없습니다. 내가 지나온 노드를 다시 찾으려면 처음부터 다시 검색해야 합니다.
2. 이중 연결 리스트
   - 특징: 노드가 앞과 뒤 두 개의 포인터를 가집니다. 양방향 이동이 가능합니다.
   - 장점: 탐색 능력이 좋습니다.
   - 단점: 포인터를 2개 저장해야 하므로 메모리를 더 많이 씁니다. Java의 LinkedList 클래스가 이 방식입니다.
3. 원형 연결 리스트
   - 특징: 마지막 노드가 Null이 아니라, 다시 처음 노드를 가리킵니다.
   - 용도: 라운드 로빈 스케줄링처럼 데이터를 계속 순환하며 처리해야 할 때 사용합니다.

## 4. 장단점

### 장점

1. 동적 크기: 데이터가 어마나 들어올지 모를 때 유용합니다. 메모리 공간을 미리 잡아둘 필요가 없습니다.
2. 효율적인 삽입/삭제: 리스트의 중간에 데이터를 넣거나 뺄 때, 물리적인 데이터 이동 없이 주소지만 바꾸면 되므로 매우 효율적입니다.

### 단점

1. 느린 접근 속도: k번째 원소를 찾으려면 앞에서부터 k번 이동해야 합니다.
2. 추가 메모리 소모: 실제 데이터 외에 주소값을 저장할 공간이 별도로 필요합니다.

반대로, 데이터 검색이 빈번하고 데이터 양이 고정적이라면 배열을 쓰는 것이 훨씬 빠릅니다.
