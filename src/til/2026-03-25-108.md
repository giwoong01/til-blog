멀티 쓰레딩(Mulit-threading)은 하나의 응용 프로그램(프로세스) 안에서 여러 개의 실행 흐름(쓰레드)이 동시에 작업을 수행하는 기술입니다.

## 1. 멀티 쓰레딩의 장점

멀티 쓰레딩은 한 번에 하나의 일만 하던 단일 스레드 방식의 한게를 극복하고, 시스템의 자원을 극한으로 끌어올리기 위해 사용됩니다.

- 자원 공유를 통한 메모리 절약: 쓰레드들은 자신이 속한 프로세스의 메모리 공간(데이터 영역, 힙 영역)을 서로 공유합니다. 프로세스를 여러 개 띄우는 것(멀티 프로세싱)보다 메모리 낭비가 훨씬 적고 효율적입니다.
- 응답성 향상: 예를 들어 웹 브라우저에서 하나의 쓰레드가 대용량 파일을 다운로드하느라 바빠도, 다른 쓰레드가 화면 스크롤이나 버튼 클릭 같은 사용자 입력을 처리할 수 있으므로 프로그램이 먹통이 되지 않습니다.
- 가벼운 문맥 교환: 프로세스 간에 작업 제어권을 넘기는 것보다 쓰레드 간에 넘기는 것이 훨씬 빠릅니다. 쓰레드끼리는 캐시 메모리나 공유 데이터 영역을 비울 필요 없이, 각자의 스택과 PC 레지스터 정도만 교체하면 되기 때문입니다.

## 2. 멀티 쓰레딩의 단점

- 동기화 문제: 여러 쓰레드가 동시에 같은 메모리 공간(공유 변수)에 접근하여 값을 읽고 수정하려고 하면, 실행 순서에 따라 결과가 뒤죽박죽이 되는 경쟁 상태(Race Condition)가 발생합니다. 앞서 설명해 드렸던 내용이 바로 이 문제입니다.
- 교착 상태(Deadlock) 위험: 동기화 문제를 막기 위해 자원에 락을 걸다 보면, 서로가 상대방이 가진 자원이 풀리기만을 영원히 기다리며 시스템이 멈춰버리는 현상이 발생할 수 있습니다.
- 디버깅의 어려움: 쓰레드의 실행 순서는 운영체제의 스케줄러가 결정하므로 매번 달라집니다. 따라서 에러가 발생핻 이를 똑같이 재현하기가 매우 까다롭습니다.

## 정리

- 멀티 쓰레딩: 하나의 프로그램 안에서 여러 작업을 동시에 처리하는 기술
- 핵심 이점: 메모리를 공유하여 자원을 아끼고, 사용자의 대기 시간을 줄여줌
- 주의점: 자원을 공유하기 때문에 발생하는 동시성 제어가 필수적임
