동일성(Identity)과 동등성(Equivalence)은 프로그래밍, 특히 Java와 같은 객체 지향 언어에서 두 객체를 비교할 때 사용되는 근본적으로 다른 두 가지 개념입니다.

- 동일성(Identity): 두 참조 변수가 메모리에서 정확히 동일한 객체(같은 주소)를 가리키고 있는지를 묻습니다. (Java의 == 연산자)
- 동등성(Equivalence): 두 참조 변수가 서로 다른 객체를 가리키더라도, 그 객체들의 논리적인 내용(값)이 같은지를 묻습니다. (Java의 .equals() 메소드)

### 동일성 (Identity) (==)

동일성은 존재 그 자체에 대한 비교입니다.

- 비교 대상: 메모리 주소 값 (참조 값)
- Java 연산자: ==
- 동작: == 연산자는 두 변수가 가리키는 메모리 주소가 물리적으로 같은지를 확인합니다.
- 결과: 두 변수가 정확히 같은 메모리 덩어리를 가리킬 때만 true를 반환합니다.
- 예외: 원시 타입 int, double, boolean 같은 원시 타입은 객체가 아니며 참조 값을 가지지 않습니다. 이 경우에는 == 연산자가 실제 값 자체를 비교합니다. (예로 int a = 10; int b = 10; 이면 a == b는 true입니다.)

### 동등성 (Equivalence) (.equals())

동등성은 논리적인 값에 대한 비교입니다.

- 비교 대상: 객체의 논리적 내용 (필드 값)
- Java 메소드: .equals()
- 동작: 이 메소드는 두 객체의 메모리 주소가 다르더라도, 개발자가 정의한 논리적인 기준에 따라 내용이 같으면 true를 반환합니다.
- Object 클래스의 기본 동작:
  - 만약 equals() 메소드를 재정의(Override)하지 않았다면, Object 클래스의 기본 equals()가 호출됩니다.
  - 이 기본 equals()는 내부적으로 == 연산자(동일성 비교)를 수행합니다.

### Java 코드 예시

```java
// 예시 1: 리터럴 방식 (String Constant Pool)
String s1 = "hello";
String s3 = "hello";

// 예시 2: new 키워드 방ㅅ기 (Heap 메모리에 새 객체 생성)
String s2 = new String("hello");

// 1. 동일성(==) 비교: 메모리 주소를 비교
System.out.println(s1 == s3); // true
// 이유: Java는 "hello"라는 리터럴을 'String Constant Pool'에 하나만 만들고
// s1과 s3가 그 '하나의' 객체 주소를 공유(재사용)하도록 합니다. (동일성 O)

System.out.println(s1 == s2); // false
// 이유: s1은 Constant Pool 주소를, s2sms 'new'를 통해 Heap 메모리에
// '새롭게 생성된' 별개의 객체 주소를 가리킵니다. (동일성 X)

// 2. 동등성 (.equals()) 비교: 논리적 내용을 비교
System.out.println(s1.equals(s2)); // true
// 이유: s1과 s2는 '다른 객체'(동일성X) 이지만,
// String 클래스는 .equals() 메소드를 "문자열의 내용이 같은지" 비교하도록
// '재정의(Override)' 해두었습니다.
// 두 객체의 내용이 "hello"로 같으므로 'true'입니다. (동등성 O)
```

### 비유

두 개념을 사람(쌍둥이)에 비유합니다.

- 동일성(==): 같은 사람인가?
  - 세상에 “나”라는 존재는 단 한명입니다.
  - 나 == 나 (true)
  - 나 == 나의 쌍둥이 동생 (false, 우리는 서로 다른 존재입니다.)
- 동등성(.equals()): 같은 유전 정보를 가졌는가?
  - equals()의 비교 기준을 유전자 정보로 재정의했다고 가정합니다.
  - 나.equals(나의 쌍둥이 동생) (true, 우리는 유전적으로 99.9% 같은 내용을 가졌습니다.)
