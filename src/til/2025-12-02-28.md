프록시는 클라이언트와 서버 사이에서 데이터를 중계하는 대리인 역할을 수행하는 서버를 의미합니다.

프록시 서버가 네트워크 구조상 어디에 위치하는가, 그리고 누구를 대리하는가에 따라 포워드 프록시(Forward Proxy)와 리버스 프록시(Reverse Proxy)로 나뉩니다.

## 포워드 프록시 (Forward Proxy)

우리가 흔히 말하는 프록시는 대부분 포워드 프록시입니다.

- 위치
  - 클라이언트(사용자)와 인터넷(외부 서버)사이
- 역할
  - 클라이언트를 대리합니다. 클라이언트가 외부 인터넷에 직접 접근하는 대신, 포워드 프록시에게 요청을 보내면 프록시가 대신 데이터를 받아와 클라이언트에게 전달합니다.

### 특징 및 용도

1. 익명성 보장 (Client IP 은닉): 서버는 요청을 보낸 주체가 클라이언트가 아닌 프록시 서버라고 인식합니다. 따라서 클라이언트의 실제 IP 주소를 감출 수 있습니다. (예: VPN)
2. 캐싱 (Caching): 사내망에서 자주 방문하는 외부 사이트의 데이터를 프록시 서버에 저장해 둡니다. 동일한 요청이 오면 외부로 나가지 않고 저장된 데이터를 반환하여 속도를 높이고 트래픽 비용을 절감합니다.
3. 콘텐츠 제한 (Access Control): 기업이나 학교 등에서 특정 사이트(예: SNS, 도박 사이트) 접근을 차단하거나, 인터넷 사용 로그를 감시하는 용도로 사용됩니다.

## 리버스 프록시 (Reverse Proxy)

백엔드 개발자에게 더 익숙하고 중요한 개념입니다.

- 위치
  - 인터넷(외부 클라이언트)과 서버(내부망)사이
- 역할
  - 서버를 대리합니다. 클라이언트가 데이터를 요청하면, 리버스 프록시가 내부망에 있는 실제 서버에 요청을 전달하고 응답을 받아 클라이언트에게 반환합니다.

### 특징 및 용도

1. 보안 (Server IP 은닉): 클라이언트는 리버스 프록시의 IP만 알 뿐, 실제 데이터를 처리하는 내부 서버의 IP는 알 수 없습니다. 이는 내부 서버를 외부의 직접적인 공격(DDoS 등)으로부터 보호하는 DMZ 역할을 합니다.
2. 로드 밸런싱 (Load Balancing): 리버스 프록시는 들어오는 요청을 뒤단에 있는 여러 대의 서버로 분산시켜 부하를 조절합니다. (예: Nginx가 요청을 받아 여러 대의 Spring Boot 서버로 분배)
3. SSL Termination: HTTPS 암호화/복호화 과정을 무거운 WAS(Spring Boot) 대신 리버스 프록시(Nginx)가 전담하게 하여, WAS의 부하를 줄여줍니다.

## 정리

- 포워드 프록시: 내가(클라이언트) 밖으로 나갈 때 나를 대신해 주는 문지기. (목적: 내 신분 감추기, 통제)
- 리버스 프록시: 밖에서 손님(클라이언트)이 들어올 때 우리 집(서버)을 대신해서 맞아주는 문지기. (목적: 집 내부 보호, 손님 안내 분산)
