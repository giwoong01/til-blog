서버 사이드 렌더링(SSR)과 클라이언트 사이드 렌더링(CSR)의 핵심적인 차이는 “최종적인 HTML 화면을 어디서 생성하느냐”에 있습니다.

이 차이는 웹 애플리케이션의 초기 로딩 속도, 검색 엔진 최적화(SEO), 그리고 서버 부하에 결정적인 영향을 미칩니다.

## 1. 서버 사이드 렌더링 (SSR, Server-Side Rendering)

SSR은 전통적인 웹 페이지 렌더링 방식이며, 서버에서 완전한 HTML을 만들어서 브라우저로 전송하는 방식입니다.

### 동작 과정

1. 사용자가 웹 사이트에 접속(요청)합니다.
2. 서버는 데이터베이스에서 데이터를 조회하고, 이를 바탕으로 HTML 파일 전체를 완성합니다.
3. 서버는 완성된 HTML 파일을 브라우저에게 응답으로 보냅니다.

4. 브라우저는 받은 HTML을 즉시 화면에 그립니다. (사용자는 이때 화면을 볼 수 있습니다.)
5. 이후 브라우저가 자바스크립트(JS) 파일을 다운로드하고 실행하여, 이미 그려진 HTML에 이벤트 리스너 등을 연결합니다. 이 과정을 ‘하이드레이션’이라고 합니다.

### 장점

- 초기 로딩 속도(FCP) 빠름: 사용자가 빈 화면을 보는 시간이 매우 짧습니다. 서버에서 이미 그려진 HTML을 받기 때문입니다.
- 검색 엔진 최적화(SEO) 유리: 검색 엔진 크롤러(Bot)가 사이트에 방문했을 때, 내용이 꽉 찬 HTML을 즉시 읽을 수 있어 정보 수집에 유리합니다.

### 단점

- 서버 부하 증가: 사용자가 클릭할 때마다 서버가 매번 HTML을 새로 만들어서 보내야 하므로 서버 리소스를 많이 사용합니다.
- 화면 깜빡임: 페이지 이동 시마다 전체 페이지를 새로고침하므로 화면이 깜빡이는 현상이 발생할 수 있습니다. (전통적인 방식의 경우)
- TTI(Time To Interactive)와 FCP의 간극: 화면은 보이지만(FCP), 자바스크립트가 완전히 로드되고 실행되기 전까지(하이드레이션 완료 전) 버튼을 눌러도 반응하지 않는 시간이 존재할 수 있습니다.

## 2. 클라이언트 사이드 렌더링 (CSR, Client-Side Rendering)

CSR은 현대적인 단일 페이지 애플리케이션(SPA, Single Page, Application)에서 주로 사용되는 방식이며, 브라우저(클라이언트)가 자바스크립트를 이용해 HTML을 동적으로 생성하는 방식입니다.

### 동작 과정

1. 사용자가 웹 사이트에 접속합니다.
2. 서버는 내용이 비어있는 뼈대 HTML과 거대한 자바 스크립트 파일을 보냅니다.
3. 브라우저는 일단 빈 화면(또는 로딩 스피너)을 띄우고 자바스크립트를 다운로드합니다.
4. 자바스크립트가 실행되면, 브라우저가 데이터를 서버에서 가져와 HTML을 생성하고 DOM에 주입하여 화면을 그립니다.

### 장점

- 초기 로딩 후 빠른 속도: 첫 로딩 때 필요한 모든 리소스를 받기 때문에, 페이지 이동 시에는 필요한 데이터(JSON)만 교체하면 되어 화면 전환이 매우 빠르고 부드럽습니다.
- 서버 부하 감소: 서버는 단순히 정적 파일과 JSON 데이터만 제공하면 되므로, 렌더링 연상을 클라이언트에 위임하여 서버 부하가 줄어듭니다.
- 사용자 경험(UX) 우수: 네이티브 앱과 유사한 자연스러운 사용자 경험을 제공합니다.

### 단점

- 초기 로딩 속도 느림: 자바스크립트 파일이 모두 다운로드되고 실행될 때까지 사용자는 빈 화면을 봐야합니다.
- SEO 불리: 검색 엔진 크롤러가 접속했을 때 HMTL이 비어있으므로(JS를 실행하지 못하는 크롤러의 경우), 내용을 수집하지 못해 검색 결과 노출에 불리할 수 있습니다. (구글 봇은 JS를 실행할 수 있으나, 여전히 SSR보다 인덱싱 속도가 느리거나 불안정할 수 있습니다.)

## 정리

- SSR 사용 권장
  - 검색 엔진 노출(SEO)이 최우선인 경우 (예: 블로그, 뉴스 사이트, e-커머스 상품 페이지)
  - 첫 화면 로딩 속도가 비즈니스에 중요한 경우
- CSR 사용 권장
  - SEO가 크게 중요하지 않은 경우 (예: 로그인 후 사용하는 어드민 페이지, 사내 시스템)
  - 풍부한 인터랙션과 앱 같은 사용성이 중요한 경우 (예: 웹 오피스, SNS 피드).

최근에는 두 방식의 장점을 결합하기 위해, 첫 페이지는 SSR로 로딩하여 SEO와 초기 속도를 잡고, 이후 상호작용은 CSR로 처리하는 방식이 표준처럼 사용되고 있습니다.
