단일 장애 지점(SPOF)은 시스템 구성 요소 중에서, 동작하지 않으면 전체 시스템이 중단되는 요소를 말합니다.

쉽게 말해 “시스템의 아킬레스건”입니다. 아무리 다른 부분이 튼튼해도 이 한 곳만 고장 나면 모든 서비스가 멈춰버리는 치명적인 약점을 의미합니다.

## 1. SPOF 예시

1. 단일 서버: 웹 서버가 딱 1대만 있는 경우, 트래픽 폭주나 하드웨어 고장으로 서버가 죽으면 서비스 전체가 다운됩니다.
2. 단일 데이터베이스: DB가 1대뿐이라면, DB 장애 시 데이터를 읽고 쓸 수 없어 모든 기능이 마비됩니다.
3. 단일 네트워크 스위치/라우터: 모든 서버가 연결된 핵심 스위치가 하나뿐이라면, 이 스위치가 고장날 경우 서버들이 멀쩡해도 통신이 불가능해집니다.
4. 전력 공급: 데이터 센터의 전원 공급 장치가 이중화 되어 있지 않다면, 정전 시 모든 장비가 꺼집니다.

## 2. 해결 방법

SPOF를 제거하는 유일한 방법은 이중화(다중화)입니다. 똑같은 역할을 하는 예비 장비를 하나 더 두는 것입니다.

- 서버 이중화: 서버를 2대 이상 두고 로드 밸런서로 트래픽을 분산시킵니다. 한 대가 죽어도 다른 서버가 요청을 처리합니다.
- DB 복제 (Replication): Master DB와 Slave DB를 나누거나 클러스터링을 통해 데이터를 실시간으로 복제해 둡니다.
- 네트워크 경로 다중화: 회선을 2개 이상의 통신사로 구성하여 하나가 끊겨도 우회할 수 있게 합니다.

## 3. 이중화 구성 방식

SPOF를 없애기 위해 장비를 2대(A, B) 준비했을 때, 운영하는 방식입니다.

- Active-Standby
  - 평소에는 A만 일하고, B는 놀면서 대기합니다.
  - A가 고장 나면 B가 즉시 투입되어 일을 이어받습니다. (Failover)
- Active-Active
  - 평소에 A와 B가 같이 일합니다. (부하 분산 효과)
  - A가 고장 나면 B가 혼자서 모든 일을 처리합니다.

## 정리

- SPOF: 얘가 죽으면 다 죽음
- 해결책: 이중화
- 목표: 고가용성 확보
