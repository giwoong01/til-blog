클래스풀 IP 주소 체계는 초기 IPv4 주소 할당 방식 중 하나로, IP 주소를 A, B, C, D, E라는 5개의 고정된 등급 으로 나누어 관리하는 방식입니다.

1971년에 도입된 방식이며, 네트워크 크기(호스트 수)에 따라 적절한 클래스를 할당하여 효율적으로 관리하려는 의도였으나, 현재는 IP 주소 낭비 문제로 인해 CIDR 방식에 밀려 더 이상 주된 할당 방식으로 사용되지 않습니다.

## 1. 네트워크 ID와 호스트ID

IP 주소는 크게 두 부분으로 나뉩니다.

- 네트워크 ID: 어떤 네트워크 그룹인지를 식별
- 호스트 ID: 그 그룹 내에서 개별 컴퓨터를 식별

클래스풀 체계는 이 네트워크 ID와 호스트 ID를 나누는 경계선이 클래스별로 고정되어 있다는 것이 핵심입니다.

## 2. 클래스별 상세 구분 (A, B, C, D, E)

주소의 맨 앞 비트(0, 10, 110…)를 보면 어떤 클래스인지 알 수 있습니다.

1. Class A (대규모 네트워크)

   - 구분 비트: 0으로 시작
   - 범위: 0.0.0.0 ~ 127.255.255.255
   - 구조: N.H.H.H (앞 비트가 네트워크, 뒤 24비트가 호스트)
   - 특징
     - 네트워크 개수는 적지만(128개), 하나의 네트워크가 가질 수 있는 호스트 수가 엄청납니다 (약 1600만개)
     - 국가 기관이나 초대형 기업 초기에 할당되었습니다.

1. Class B (중규모 네트워크)

   - 구분 비트: 10으로 시작
   - 범위: 128.0.0.0 ~ 191.255.255.255
   - 구조: N.N.H.H (앞 16비트가 네트워크, 뒤 16비트가 호스트
   - 특징
     - 하나의 네트워크당 약 65000개의 호스트를 가질 수 있습니다.
     - 대학교나 중대형 기업에서 사용했습니다.

1. Class C (소규모 네트워크)

   - 구분 비트: 110으로 시작
   - 범위: 192.0.0.0 ~ 223.255.255.255
   - 구조: N.N.N.H (앞 24비트가 네트워크, 뒤 8비트가 호스트)
   - 특징
     - 네트워크 개수는 아주 많지만, 하나의 네트워크당 호스트 수가 적습니다.
     - 소규모 회사, 개인, 카페 등에서 흔히 볼 수 있는 192.168… 대역이 여기에 속합니다.

1. Class D & E (특수 목적)
   - Class D (1110): 멀티 캐스트용 주소입니다(224.0.0.0 ~ 239.255.255.255)
   - Class E (1111): 미래를 위한 연구용/실험용으로 예약된 주소입니다. (240.0.0.0)

## 3. 클래스풀의 한계

이 체계는 치명적인 단점 떄문에 현재는 사실상 폐기되었습니다. 바로 주소 낭비 문제입니다.

[문제상황]

어떤 중소기업이 컴퓨터 300대를 쓴다고 가정해 봅시다.

1. Class C를 주자니: 최대 254개밖에 못 써서 부족합니다.
2. Class B를 주자니: 65534개를 쓸 수 있는데, 300개만 쓰고 나머지 65200여 개는 버려지게 됩니다.

이런 식으로 할당 단위가 너무 경직되어 있어 IP 주소가 빠르게 고갈되었습니다.

해결책: CIDR

이를 해결하기 위해 등장한 것이 CIDR와 서브넷 마스크입니다. 클래스라는 고정된 틀을 깨버리고, 필요한 만큼 비트 단위로 잘라서 (192.168.10.0/23 처럼) 할당하는 방식입니다.

## 정리

- 클래스풀: IP를 A, B, C 등급으로 나누어 네트워크 크기를 고정한 방식
- 문제점: 중간 크기의 조직에 할당할 때 IP 낭비가 너무 심함
- 현재: 유연하게 자르는 클래스리스(CIDR) 방식을 사용함
