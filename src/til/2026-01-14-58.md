세 가지 개념의 핵심적인 관계는 “URI는 URL과 URN을 포함하는 상위 개념이다”라는 것입니다.

쉽게 비유하자면 다음과 같습니다.

- URI: 사람을 구별하는 방법 (포괄적 개념)
- URL: 사람의 집주소 (위치로 찾기)
- URN: 사람의 주민등록 번호 (이름으로 찾기)

## 1. URI (Uniform Resource Identifier)

“통합 자원 식별자”

인터넷에 있는 자원을 나타내는 유일한 주소를 의미합니다. URL과 URN은 URI의 하위 개념입니다. 즉, 모든 URL은 URI이지만, 모든 URI가 URL인 것은 아닙니다.

## 2. URL (Uniform Resource Rocator)

“통합 자원 위치”

자원이 어디에 있는지와 어떻게 접근할지를 명시합니다. 우리가 웹 브라우저 주소창에 치는 주소는 99% URL입니다.

- 특징
  - 접근에 필요한 프로토콜이 포함됩니다
  - 자원의 위치가 바뀌면 URL은 더 이상 유효하지 않게 됩니다.
- 형식: 프로토콜://도메인(IP)/경로
- 예시: https://www.google.com/images/logo.png

## 3. URN (Uniform Resource Name)

“통합 자원 이름”

자원의 위치와 상관 없이, 자원에 부여된 고유한 이름입니다.

- 특징
  - 자원의 위치가 바뀌어도 URN은 변하지 않습니다.
  - 프로토콜(http)을 포함하지 않습니다.
  - 실제 자원을 찾으려면 URN을 URL로 변환해 주는 시스템이 필요하기 때문에, 웹 브라우저에서 직접적으로 잘 쓰이지는 않습니다.
- 형식: urn:스키마:식별자
- 예시: urn:isbn:9788960773417 등

## 정리

개발자들 사이에서는 엄격하게 구분하기 보다, API 요청 주소 등을 말할 때 통상적으로 URL이라고 부르는 경우가 많습니다. 하지만 REST API 설계 문서 등에서는 리소스를 식별한다는 의미를 강조하기 위해 URI 라는 용어를 사용하는 것이 더 정확합니다.
