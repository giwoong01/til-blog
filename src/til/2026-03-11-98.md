트라이는 문자열의 집합을 저장하고 탐색하는 데 특화된 트리 기반의 자료구조입니다.

문자열의 접두사(Prefix)를 활용하여 탐색을 진행하기 때문에 접두사 트리라고도 불리며, 정보 검색이라는 단어에서 그 유래가 시작되었습니다.

## 1. 구조 및 동작 방식

트라이는 문자열을 구성하는 각각의 문자를 노드 간의 간선으로 취급하여 트리를 구성합니다.

- 루트 노드: 트라이의 시작점이며, 특정 문자를 포함하지 않고 비어있는 상태로 자식 노드들을 가리키는 포인터만 가집니다.
- 자식 노드 포인터: 각 노드는 이어질 수 있는 문자의 수만큼 포인터 배열이나 해시맵을 가집니다. (예를 들어, 영어 소문자만 다룬다면 각 노드는 최대 26개의 자식 포인터 배열을 가집니다.)
- 단어의 끝 (isEndOfWord): 특정 경로까지의 문자 조합이 하나의 완성된 단어인지를 판별하기 위해, 각 노드 내부에 boolean 타입의 플래그를 저장합니다.
  - 예를 들어 “CAR”와 “CARD”가 모두 저장되어 있다면, ‘R’노드와 ‘D’ 노드 모두 isEndOfWord = true 상태가 됩니다.

## 2. 복잡도

트라이의 가장 큰 특징은 속도와 메모리의 트레이드오프입니다.

- 시간 복잡도: O(L)
  - L은 찾고자 하는 문자열의 길이입니다.
  - 트라이에 저장된 전체 단어의 개수와 무관하게, 찾으려는 문자열의 길이만큼만 노드를 순회하면되므로 검색 속도가 매우 빠릅니다. 이진 탐색 트리(BST)의 O(L logN)보다 훨씬 우수합니다.
- 공간 복잡도: O (N x M)
  - N은 총 노드의 수, M은 문자 집합(예: 알파벳 26개)의 크기입니다.
  - 자식 노드가 실제로 존재하지 않더라도, 포인터 배열을 미리 할당해 두어야 하는 구조적 특성 때문에 메모리 소모가 매우 심하다는 것이 가장 큰 단점입니다.

## 3. 사용처

빠른 문자열 탐색이 필요하고, 메모리 여유가 있는 시스템에서 주로 활용됩니다.

1. 자동 완성: 포털 사이트 검색창에 “app”을 치면, “apple”, “application” 등이 뜨는 것처럼, 특정 접두사로 시작하는 모든 단어를 빠르게 찾을 때 사용합니다.
2. 맞춤법 검사기: 입력된 단어가 사전에 등록된 올바른 단어인지(경로가 이어지면 isEndOfWord가 true인지) 검증할 때 사용합니다.
3. 네트워크 라우팅: 라우터에서 목적지 IP 주소를 찾을 때, 가장 길게 일치하는 경로를 찾는 Longest Prefix Matching 알고리즘에 응용됩니다.

## 정리

트라이는 문자열 검색에 있어 길이(L)에 비례하는 극한의 속도를 보장하지만, 그만큼 막대한 메모리를 희생하는 자료구조입니다.
