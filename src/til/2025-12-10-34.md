HTTPS는 웹 브라우저와 웹 서버 간의 데이터를 암호화하여 주고받는 보안 통신 프로토콜입니다.

기존의 HTTP는 데이터를 평문으로 전송하기 때문에, 네트워크 상에서 누군가가 패킷을 가로채면 비밀번호나 카드 정보 같은 민감한 데이터를 그대로 볼 수 있다는 치명적인 보안 취약점이 있었습니다. HTTPS는 이를 해결하기 위해 HTTP 하부에 SSL 또는 그 계승자인 TSL라는 보안 계층을 추가하여 만들어졌습니다.

## 1. HTTPS의 3대 핵심 보안 목표

HTTPS는 단순히 데이터를 암호화하는 것 이상의 세가지 중요한 보안 기능을 제공합니다.

1. 기밀성 (Confidentiality - 암호화): 데이터를 암호화하여 전송합니다. 해커가 중간에서 데이터를 가로채더라도 암호화된 난수만 보이므로 내용을 해독할 수 없습니다.
2. 무결성 (Integrity - 데이터 위변조 방지): 데이터가 전송되는 도중에 제3자에 의해 수정되거나 변조되지 않았음을 보장합니다. 전송된 데이터에 디지털 서명(MAC) 등을 포함하여 수신 측에서 위변조 여부를 검증합니다.
3. 인증 (Authentication - 시원 증명): 사용자가 접속한 사이트가 가짜 사이트가 아니라 진짜 신뢰할 수 있는 서버임을 보장합니다. 이는 신뢰할 수 있는 기관인 CA가 발급한 SSL 인증서를 통해 이루어집니다.

## 2. 암호화 방식: 하이브리드 암호화 시스템

HTTPS는 속도와 보안의 균형을 맞추기 위해 대칭키 암호화와 비대칭키(공개키) 암호화 방식을 혼합하여 사용합니다.

- 대칭키 암호화 (Symmetric Encryption)
  - 암호화할 때와 복호화할 때 동일한(대칭키)를 사용합니다.
  - 장점: 연산 속도가 매우 빠릅니다.
  - 단점: 통신 당사자끼리 이 키를 안전하게 공유하는 것이 어렵습니다.
- 비대칭키 암호화 (Asymmetric Encryption)
  - 공개키와 개인키라는 두 개의 키 쌍을 사용합니다. 공개키로 암호화하면 개인키로만 풀 수 있고, 개인키로 암호화하면 공개키로만 풀 수 있습니다.
  - 장점: 키 배송 문제가 해결되며, 인증(개인키 소유 증명)이 가능합니다.
  - 단점: 연산 속도가 대칭키보다 현저히 느립니다(CPU 부하).

결론적으로 HTTPS는

1. 처음 연결을 맺을 때는 안전한 키 교환을 위해 비대칭키 암호화를 사용하고,
2. 실제 대용량 데이터를 주고받을 때는 속도를 위해 대칭키 암호화를 사용합니다.

## 3. HTTPS 동작 과정 (SSL/TSL Handshake)

브라우저가 HTTPS 사이트에 접속할 때, TCP 연결(3-way Handshake)이 맺어진 직후 SSL/TSL Handshake가 수행됩니다. 이 과정은 보안 채널을 생성하기 위한 협상 단계입니다.

1. Client Hello: 브라우저가 서버에 접속하며 자신이 지원하는 암호화 방식 목록과 무작위 데이터를 보냅니다.
2. Server Hello: 서버는 브라우저가 보낸 목록 중 가장 강력한 암호화 방식을 선택하고, 자신의 무작위 데이터를 보냅니다. 그리고 자신의 신원을 증명할 SSL 인증서를 브라우저에 전송합니다.
3. 인증서 검증: 브랑줘는 내장된 신뢰할 수 있는 CA 목록을 통해 서버의 인증서가 유효한지 검증합니다. (위조된 경우 경고창을 띄웁니다.)
4. Pre-Master Secret 생성 및 암호화: 브라우저는 서버의 공개키를 사용하여, 통신에 사용할 대칭키의 재료가 되는 정보(Pre-Master Secret)를 암호화해서 서버에 보냅니다.
5. 세션 키(Session Key) 생성: 이제 브라우저와 서버는 서로 교환한 무작위 데이터와 Pre-Master Secret을 조합하여, 실제 통신에 사용할 동일한 세션 키(대칭키)를 생성합니다. (이 키는 외부로 유출되지 않았습니다.)
6. 보안 통신 시작: 핸드셰이크가 종료되고, 이후의 모든 HTTP 요청과 응답은 이 세션 키로 암호화되어 전송됩니다.

## 4. 정리

HTTPS는 HTTP에 SSL/TSL 보안 계층을 얹은 것입니다.

1. CA 인증서를 통해 접속하려는 서버가 진짜임을 인증합니다.
2. 비대칭키 암호화를 사용해 대칭키를 안전하게 교환합니다.
3. 실제 데이터 전송은 대칭키를 사용해 빠르게 암호화하여 처리합니다.

이로써 도청, 변조, 위장으로부터 안전한 인터넷 통신 환경을 제공합니다.
