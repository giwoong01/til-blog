동시성과 병렬성은 다중 작업을 처리하는 방식에 대한 개념으로, 컴퓨터 과학에서 매우 중요하지만 자주 혼동되는 용어입니다.

핵심적인 차이는 ‘동시에 처리되는 것처럼 보이는가(동시성)’와 ‘진짜로 동시에 처리되는가(병렬성)’에 있습니다.

## 1. 동시성 (Concurrency)

“하나의 코어에서 여러 작업을 번갈아 가며 빠르게 처리하여 동시에 실행되는 것처럼 보이게 하는 것.”

- 정의
  - 어려 작업이 시간을 나누어(Time-Slicing) 실행되면서 서로 상태를 공유하고 진행되는 것처럼 보이게 하는 방식입니다.
- 하드웨어 요구 사항
  - 단일 코어 환경에서도 구현 가능합니다.
- 동작 방식
  - CPU 코어는 한 번에 하나의 작업만 처리할 수 있지만, 운영체제의 스케줄러가 각 작업에 아주 짧은 시간을 할당하고, 이 작업을 매우 빠르게 교체(Context Switching)합니다.
- 목적
  - 작업 간의 상호작용 및 관리에 초점을 맞춥니다. 주로 IO 작업(네트워크 통신, 디스크 입출력)처럼 CPU가 쉬어야 하는 시간이 많은 작업의 효율을 극대화하는 데 사용됩니다.
- 예시
  - 싱글 코어 CPU가 웹 브라우징과 음악 재생을 동시에 처리하는 것처럼 보이는 상황

## 2. 병렬성

“두 개 이상의 코어에서 여러 작업을 동시에 처리하여 실제로 동시에 실행되게 하는 것.”

- 정의
  - 여러 작업이 물리적으로 분리된 다수의 처리 장치(코어)에서 동시에 실행되는 방식입니다.
- 하드웨어 요구 사항
  - 멀티 코어 이상의 환경이 필수적입니다.
- 동작 방식
  - 각 코어가 완전히 독립적으로 서로 다른 작업을 전담하여 처리합니다.
- 목적
  - 작업의 처리 속도 자체를 높이는 데 초점을 맞춥니다. 주로 복잡한 연산(행렬 계산, 데이터 처리)처럼 CPU 자원을 많이 사용하는 작업에 사용됩니다.
- 예시
  - 쿼드 코어 CPU에서 4개의 스레드가 각기 다른 연산을 동시에 수행하는 상황.

## 정리

두 개념은 상호 배타적이지 않으며, 상위 개념인 ‘동시성’안에 ‘병렬성’이 포함될 수 있습니다.

- 멀티 코어 환경: 동시성과 병렬성을 모두 달성할 수 있습니다. (예: 4개의 작업을 4개의 코어에서 동시에 실행하면 병렬성, 8개의 작업을 4개의 코어에서 번갈아 가며 실행하면 동시성도 함께 달성
- 싱글 코어 환경: 오직 동시성만 달성할 수 있습니다. 아무리 빨리 교체해도 물리적으로는 한 순간에 하나의 작업만 실행되기 때문입니다.

결국 동시성은 ‘작업 관점’의 문제이고, 병렬성은 ‘하드웨어 관점’의 문제입니다.
